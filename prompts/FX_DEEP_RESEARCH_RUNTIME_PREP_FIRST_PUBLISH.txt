# FX Deep Research Runtime — Prep-First Production Publish Flow

This file is the dedicated **prep-first production runtime** for the Weekly FX Review.

Use this file when you want the canonical ChatGPT-first production path that must end in a new report written to `output/` and then handed to the FX delivery workflow.

---

## Core operating rule

This runtime must **not** start immediately with deep research.

Before doing any research or writing any report, you must treat the technical/state refresh as a mandatory prerequisite.

The required sequence is:

1. confirm that prep ran successfully
2. confirm that the refreshed state commit landed on `main`
3. confirm that manifest, overlay, and portfolio state reconcile to the same fresh state window
4. only then begin the deep-research run
5. generate the Weekly FX Review in markdown
6. write the markdown file to `output/weekly_fx_review_YYMMDD.md` or `_NN.md`
7. leave render / email / delivery confirmation to the downstream FX GitHub workflow

If the prep step was not run first, or if the refreshed state is stale or incoherent, the ChatGPT run must be treated as blocked.

## Research objective

After the prep step has completed and the required files have been read, produce a **full Weekly FX Review** that preserves the existing:
- methodology
- research logic
- policy-divergence logic
- state-authority rules
- technical-overlay role as confirmation rather than primary engine
- portfolio-engine expectations
- report structure
- executive look and feel
- delivery readiness

The report must feel like the same premium family as the existing FX output standard.

---

## Fresh research requirement

Once prep is complete, begin a **fresh deep-research pass**.

At minimum, refresh:

### A. Global macro regime
- growth backdrop
- inflation backdrop
- labor-market direction where relevant
- risk-on / risk-off regime
- energy / commodity backdrop
- trade-policy backdrop
- major market stressors and second-order effects

### B. Central banks and policy divergence
Refresh the current stance / latest relevant decision / reaction-function framing for:
- Fed
- ECB
- BoE
- BoJ
- SNB
- BoC
- RBA
- RBNZ
- Banxico when MXN is in scope

### C. Geopolitics and event risk
Refresh the active geopolitical and event-risk layer relevant to FX allocation, including:
- war / conflict escalation or de-escalation
- energy-shock implications
- sanctions / tariff / trade-policy developments
- other cross-border macro stressors that matter for FX regime selection

### D. Current market-validating facts
Refresh only the facts needed to support strategic judgment.
Do not turn the report into a random news digest.
Research must serve portfolio judgment.
